# Задача

Написать краткий конспект по теме, написанной в названии MD-файла. Конспект будет использоваться для Obsidian.

# Структура

Структура конспекта будет отличаться на ДО и ПОСЛЕ твоего преобразования. Структура, которая будет ДО:

- `Refs` - статьи, книги, ссылки и прочее;
- `API` - классы, методы, gems, libraries, modules, functions, DSL-элементы;
- `Questions` - вопросы, на которые нужно ответить отдельными topic
- `Topics` - дополнительные темы, которые нужно раскрыть
- `Raw Info`  - основная информация для конспекта

Стуктура конспекта, которая должна быть ПОСЛЕ:

- `hzhz`

То что должно войти в итоговый конспект: 

1. Ref - статьи, ссылки на книги и прочее (желательно делать ссылками, если такое сделать невозможно, просто текст) 
2. API - классы, методы, гемы и прочее, про которые здесь будет объяснение. Можно добавить ссылку на апи методов/классов. Будет добавляться если речь идет о программировании. Можно разделить на следующие подблоки: classes, methods, gems или libraries (в зависимости от языка)
3. Definition - Блок с определением + отдельным обзацем какую проблему решает тот или иной инструмент 
4. Topics - темы, про которые можно рассказать. 
5. Best Practices - лучшие практики и трюки с этим инструментом 

Как будет приходить сжатый план конспекта (помимо описанных выше блоков, эти блоки будут нужны для развития конспекта): 
1. Questions - вопросы, на которые нужно дать ответ в конспекте в отдельном topic
2. API - методы, классы, гемы и прочее, которые нужно добавить в API
3. Topics - темы, которые нужно добавить в Topics 
4. Raw Info - это будет основная информация конспекта, взятая из статей. Ее нужно будет переписать на русский язык и разбить на темы. Там также будет храниться определение и проблема
