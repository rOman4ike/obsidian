# Задача

Написать краткий конспект по теме из названия Markdown-файла для Obsidian.

Верни только готовый `Markdown` без пояснений.

---

# Входные данные

Во входной заметке могут быть блоки:

- `Refs` — статьи, книги, документация, ссылки и другие источники;
- `API` — programming concepts, API, patterns, caveats;
- `Questions` — вопросы, возникшие в процессе изучения темы;
- `Additional Topics` — дополнительные темы, которые нужно раскрыть;
- `Raw Info` — основная информация для конспекта.

---

# Результат

Итоговая заметка должна иметь структуру:

- `TL;DR`
- `Refs`
- `API`
- `Topics`
- `Research Questions`

Начало заметки:

```markdown
> [!info] TL;DR
> {{ краткое резюме только на основе входных данных }}
```

# Правила преобразования

## Refs

- Сохрани источники из входных данных.
- Если ссылок нет, можно добавить внешние ссылки и пометить их (внешний источник).

## API

- Сохрани только те programming concepts, API, patterns и caveats, которые есть во входных данных.
- Сохрани ссылки из входных данных.
- Если ссылок нет, можно добавить внешние ссылки и пометить их (внешний источник).

## Topics

- Формируй только на основе Raw Info и Additional Topics.
- Не добавляй новые темы от себя.
- Для каждого topic обязательно создай блоки:
    - Definition
    - How it Works
    - Usage
    - Examples
    - When to Use
    - Summary
- Опционально можно добавить Best Practices.

## Research Questions

- Перенеси сюда все вопросы из Questions.
- Для каждого вопроса дай краткий ответ.
- Для каждого ответа обязательно укажи источник.
- Используй не более двух источников на один ответ.
- Если источник внешний, помечай его (внешний источник).
- Если надежный ответ найти не удалось, напиши:  
    Ответ не найден, требуется дополнительная проверка источников.

---

# Callouts

Если информации из входных данных недостаточно для заполнения блока в Topics, вставь:

`> [!warning] > Требуется проверка информации`

---

# Ограничения

## Разрешено

- Переписывать текст на русский язык;
- Сокращать;
- Структурировать;
- Использовать внешние источники только там, где это явно разрешено;
- Оставлять термины на английском языке.

## Запрещено

- Выдумывать;
- Писать факты без опоры на входные данные или источник;
- Придумывать примеры, которых нет во входных данных или источниках;
- Подменять терминологию;
- Переводить термины, названия API, classes, methods, gems, libraries.
