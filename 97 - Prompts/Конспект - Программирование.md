# Задача

Написать краткий конспект по теме, написанной в названии MD-файла. Конспект будет использоваться для Obsidian.

# Правила

## Структура

Структура конспекта будет отличаться на ДО и ПОСЛЕ твоего преобразования. 

Структура, которая будет ДО:
- `Refs` - статьи, книги, ссылки и прочее;
- `API` - Programming concepts (API, patterns and caveats).. Можно добавить ссылку на апи методов/классов
- `Questions` - вопросы, на которые нужно ответить отдельными topic
- `Topics` - дополнительные темы, которые нужно раскрыть
- `Raw Info`  - основная информация для конспекта

Стуктура конспекта, которая должна быть ПОСЛЕ:
- `Refs`
- `API`
- `Topics`
- `Best Practices`
- `Summary`

Для блоков `Refs` и `API` список должен быть дополнен ссылками на источник.

В `Topics` ОБЯЗАТЕЛЬНО быть следующие блоки:
- `Definition`
- `How it Workds`
- `Usage`
- `Examples`
- `When to Use`
- `Summary`

В `Topics` ОПЦИОНАЛЬНО быть следующие блоки:
## Callouts

Если информация не содержится во входных данных, но без нее блок нельзя корректно заполнить, пиши:

```Markdown
> [!warning]  
> Требуется проверка информации
```

Если информации недостаточно для создания `topic`, все равно создай этот `topic` и напиши:  

```Markdown
> [!warning]  
> Требуется проверка информации
```

## Разрешено

- Переписывать текст на русский язык;
- сокращать;
- структурировать;
- Делать только те выводы, которые прямо и однозначно следуют из входных данных.
- Термины должны быть на английском языке

## Запрещено

- Выдумывать;
- Писать то, чего нет в `Raw Info` или `Questions`
- Придумывать примеры, которых нет во входных данных;
- Подменять терминологию;
- Переводить термины, названия API, classes, methods, gems, libraries.
