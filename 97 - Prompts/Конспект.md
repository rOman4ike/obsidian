# Задача

Написать краткий конспект по теме, написанной в названии MD-файла. Конспект будет использоваться для Obsidian. Верни только готовый `Markdown` без пояснений.

---

# Правила

## Структура

Структура конспекта будет отличаться на ДО и ПОСЛЕ твоего преобразования. 

В блоках `Refs` и `API` сохрани ссылки на источники, если они присутствуют во входных данных. Если они отсутствуют попытайся найти свои и пометь такие ссылки `(внешний источник)`
### На входе

- `Refs` - статьи, книги, документация, ссылки и другие источники;
- `API` - programming concepts, API, patterns, caveats;
- `Questions` - вопросы, возникшие в процессе изучения темы;
- `Additional Topics` - дополнительные темы, которые нужно раскрыть;
- `Raw Info` - основная информация для конспекта.

### На выходе

- `TL;DR` - краткое резюме только на основе входных данных;
- `Refs` - статьи, книги, документация, ссылки и другие источники из входных данных;
- `API` - Programming concepts (API, patterns and caveats);
- `Topics` - темы конспекта, формируются на основе блоков `Additional Topics` и `Raw Info`;
- `Research Questions` - нужен для отдельной фиксации вопросов, возникших в процессе изучения темы, и кратких ответов на них.

Заметка должна начинаться `TL;DR`. Шаблон:

```Markdown
> [!info] TL;DR  
> {{ краткое резюме только на основе входных данных }}
```

## Правила преобразования

- `Topics` формируются только на основе `Raw Info` и `Additional Topics`.
- `Questions` не нужно автоматически преобразовывать в `Topics`.
- Все вопросы из `Questions` переноси в блок `Research Questions`.
- Для каждого вопроса в `Research Questions` дай краткий ответ.
- Для каждого ответа обязательно укажи источник.
- Для каждого ответа используй не более двух источников.
- Если источник не содержится во входных данных, помечай его как `(внешний источник)`.
- Если по вопросу нельзя найти надежный ответ, так и напиши: `Ответ не найден, требуется дополнительная проверка источников.`

## Topics

В `Topics` ОБЯЗАТЕЛЬНО будут следующие блоки:
- `Definition`
- `How it Works`
- `Usage`
- `Examples`
- `When to Use`
- `Summary`

В `Topics` ОПЦИОНАЛЬНО будут следующие блоки:
- `Best Practices`

## Callouts

Если информации из входных данных недостаточно для заполнения блока в `Topics`, пиши:

```Markdown
> [!warning]  
> Требуется проверка информации
```

# Оформление

## Разрешено

- Переписывать текст на русский язык;
- сокращать;
- структурировать;
- Делать только те выводы, которые прямо и однозначно следуют из входных данных.
- Использовать внешние источники только там, где это явно разрешено;
- Термины должны быть на английском языке

## Запрещено

- Выдумывать;
- Писать то, чего нет во входных данных. Исключение: блок `Research Questions`, для него ссылаться на внешние источники можно
- Придумывать примеры, которых нет во входных данных;
- Подменять терминологию;
- Переводить термины, названия API, classes, methods, gems, libraries.
